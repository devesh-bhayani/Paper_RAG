Run the retrieval eval before declaring any retrieval/schema change done.

## Conventions

- **All config lives in `config.py`** â€” no YAML, no env-var config, no scattered
  constants. Profile switch (`pc`/`mac`) selects models, context sizes, thermal pacing.
- One module per concern in `ragcore/` (ingest / embed / store / generate / present /
  jobs). Heavy imports (docling) happen lazily inside functions.
- Generation functions return `(token_stream, chunks)` tuples; callers accumulate.
- **Citation contract**: prompt numbers chunks 1..k in list order; model cites `[n]`;
  UI/tests resolve `[n]` against the same list. Never reorder between build and display.
- `doc_id` = posix-style path relative to `data/library/` (always forward slashes,
  minted via `.as_posix()` â€” keep it that way; the index moves between OSes).
- Deliberate shortcuts carry a `ponytail:` comment naming the ceiling and upgrade path.
  Keep doing this; don't "fix" them without reading the comment.
- Each new feature ships one runnable gate script in `tests/`, assert-based, no framework.

## Gotchas

- **Gradio must stay `>=5,<6`.** v6 renamed ChatInterface/launch kwargs and the chat
  panel rendered blank. Don't upgrade casually.
- **Don't "simplify" the torch block in `pyproject.toml`.** torch/torchvision are
  pinned to the pytorch **cu130** index with win32-only resolution. Plain PyPI torch on
  Windows is CPU-only â†’ Docling parsing gets ~6Ă— slower. cu128 lacks cp313 win wheels.
- **Never change `EMBED_MODEL` (bge-m3) or `store.SCHEMA` without**: wipe
  `data/lancedb` â†’ re-ingest (fast â€” staging cache) â†’ re-run `eval_retrieval.py`.
- **`FULLDOC_NUM_CTX` on pc is 16384 and measured.** 20480/24576 spill to CPU on the
  8 GB card (`ollama ps` shows the split). Don't raise it; the mac profile already has 32768.
- Ollama env vars (`OLLAMA_FLASH_ATTENTION=1`, `OLLAMA_KV_CACHE_TYPE=q8_0`,
  `OLLAMA_MAX_LOADED_MODELS=1`) are Windows user-level; **restart the Ollama tray app**
  after changing. Sudden slow decode? Check `ollama ps` for CPU spill first.
- All qwen3 calls pass `think=False`. Removing it leaks thinking traces and slows turns.
- Run entrypoints **from the repo root**; background shells may start elsewhere â€”
  `Set-Location` first (a real failure happened this way).
- First run after a fresh clone needs network once (Docling models, bge-m3 tokenizer
  from HF); offline after that.
- Windows console is cp1252 â€” print model output only after
  `sys.stdout.reconfigure(encoding="utf-8")` (see `generate.main`).
- Re-uploading a same-named PDF is *skipped* (idempotent by doc_id); there is no
  delete/re-index path yet (GAPS.md #8).

## Rules

- `data/library/` is the user's source PDFs â€” **never delete**. `data/lancedb`,
  `data/staging`, `data/exports` are rebuildable. All of `data/` + `.venv` are gitignored.
- `uv.lock` is generated by `uv sync` â€” never hand-edit.
- `present.DECK_SYSTEM` encodes the professor's **graded** format (section arc
  Introduction â†’ Background â†’ Methodology â†’ Evaluation â†’ Your Thoughts & Discussion;
  rubric 3/2/3/2; Methodology weighted heaviest; HANDOFF markers for 3 presenters).
  Don't reword its structural requirements; tests assert the section names.
- No LangChain/LlamaIndex/orchestration frameworks â€” direct library calls only.
- Quality upgrades (reranker, query rewriting, etc.) are banned until
  `eval_retrieval.py` shows a deficit (currently 20/20).
- `gpu_lock` (in `jobs.py`) must wrap anything that hits the LLM or embedder from the
  UI â€” one 8 GB GPU, one resident model.
- Mac migration (~Aug 2026): flip `PROFILE` in config.py, replace the pyproject torch
  block, pull mac-tier models. (doc_id separators already posix â€” GAPS.md #1 fixed.)
